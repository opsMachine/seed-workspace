# z-archived/

Superseded versions of strategy artifacts kept for historical trail.

When a major artifact gets restructured (`strat-summary.md` v1 → v2, `pyramid.excalidraw` v3 → v4, etc.), save the prior version here as `<name>-vN-YYYY-MM-DD.<ext>` so the lineage is preserved.

Also where `Strategy/to integrate/` source docs land after the [`integrate`](../../.claude/skills/integrate/SKILL.md) skill folds their content into canonical.

The `z-` prefix sorts this folder to the bottom in most file browsers — keeps it out of the way without fully hiding it.

## When to archive vs. delete

- **Archive** when the prior version contains reasoning trail you might want to revisit ("why did we hold X?")
- **Delete** when the content is fully redundant with what replaced it and there's no useful reasoning trail

When in doubt, archive. Disk is cheap; lost reasoning is expensive.
